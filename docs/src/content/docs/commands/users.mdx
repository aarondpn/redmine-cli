## list

```bash
redmine users list [flags]
```

| Flag | Description |
|------|-------------|
| `--status`     | Filter by status: `active`, `registered`, `locked` |
| `--name`       | Filter by name or login |
| `--group`      | Filter by group name or ID |
| `--limit`      | Maximum number of results |
| `--offset`     | Result offset for pagination |
| `-o, --output` | Output format |

## get

```bash
redmine users get <id-or-name> [flags]
```

Accepts a numeric ID, login, full name, or `me`.

## me

```bash
redmine users me [flags]
```

Shows the authenticated user's details ‚Äî useful in scripts to resolve the current account.

## create

```bash
redmine users create [flags]
```

| Flag | Description |
|------|-------------|
| `--login`     | Login name ‚Äî <Badge text="required" variant="danger" /> |
| `--password`  | Password ‚Äî <Badge text="required" variant="danger" /> |
| `--firstname` | First name ‚Äî <Badge text="required" variant="danger" /> |
| `--lastname`  | Last name ‚Äî <Badge text="required" variant="danger" /> |
| `--mail`      | Email address ‚Äî <Badge text="required" variant="danger" /> |
| `--admin`     | Grant admin privileges |

<Aside type="caution" title="Passwords on the command line">
  Passing `--password` on the command line may leak the value to shell history. Prefer creating the account in Redmine UI, or pipe the value from a secure source.
</Aside>

## update

```bash
redmine users update <id-or-name> [flags]
```

Accepts the same flags as [`create`](#create) (all optional). The user argument accepts a numeric ID, login, full name, or `me`.

## delete <Badge text="destructive" variant="caution" />

```bash
redmine users delete <id-or-name> [flags]
```

| Flag | Description |
|------|-------------|
| `-f, --force` | Skip confirmation prompt |
