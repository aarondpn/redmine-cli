redmine-cli 针对 AI 代理提供两种集成方式，具体取决于你的工具如何与外部软件交互。

- **代理 Skill**：代理加载的一组指令，用于高效驱动 `redmine` CLI。适用于通过 shell 调用 CLI 的编程代理。
- **MCP 服务器**：`redmine mcp serve` 通过 [Model Context Protocol](https://modelcontextprotocol.io) 将 CLI 暴露为 MCP 服务器。适用于通过 MCP 发现工具的宿主。

按你的宿主环境选用即可，也可以两种都安装。

## 代理 Skill

该 skill 教给代理的是 `--help` 无法传达的内容：输出格式、分页、过滤、名称解析以及常用工作流。安装之后，代理便知道要使用 `-o json`、在值存在歧义时先查询以解析、并无需猜测即可选择正确的标志。

### 安装

<Tabs syncKey="skill-scope">
  <TabItem label="全局">
    对本机所有项目生效。

    ```bash
    redmine install-skill --global
    ```
  </TabItem>
  <TabItem label="项目本地">
    仅对当前工作目录生效。

    ```bash
    redmine install-skill
    ```
  </TabItem>
</Tabs>

<Aside type="note">
  底层使用 [skills.sh](https://skills.sh) 安装器（`npx skills add`），需要将 Node.js 添加至 `PATH`。
</Aside>

### 手动配置

如果不想运行安装器，可将 [`skills/redmine-cli/SKILL.md`](https://github.com/aarondpn/redmine-cli/blob/main/skills/redmine-cli/SKILL.md) 的内容复制到代理的指令文件中，然后重启代理。

该文件同时也是 skill 所包含内容的权威来源。若想确切了解该 skill 编码了哪些行为，请直接阅读此文件。

## MCP 服务器

`redmine mcp serve` 通过 stdio 将 CLI 暴露为一个 [Model Context Protocol](https://modelcontextprotocol.io) 服务器。支持 MCP 的宿主可通过工具调用驱动 Redmine，复用与其他所有 `redmine` 命令相同的基于配置文件的身份验证。

### 工作原理

- **传输方式：** stdio。宿主生成 `redmine mcp serve` 进程，并通过其标准流使用 JSON-RPC 通信。
- **身份验证：** 默认使用当前活跃的配置文件。可通过 `--profile <name>`、`--server` / `--api-key` 或 `REDMINE_*` 环境变量覆盖，与其他子命令完全一致。
- **默认只读。** 修改类工具（创建 / 更新 / 删除、评论、分配、关闭、重新打开）仅在传入 `--enable-writes` 时才会注册。未传入该标志时，这些工具不会出现在 `tools/list` 中。

<Aside type="caution">
  写入工具具有破坏性。除非宿主提供了你信任的逐次审批界面（per-call approval UI），否则建议保持禁用。
</Aside>

### 前置条件

`redmine` 必须位于宿主的 `PATH` 中，且已有配置文件完成登录（`redmine auth login`）。无需 Node.js。

### 宿主配置

<Tabs syncKey="mcp-host">
  <TabItem label="Claude Desktop">
    编辑 `claude_desktop_config.json`：

    - **macOS：** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows：** `%APPDATA%\Claude\claude_desktop_config.json`

    ```json
    {
      "mcpServers": {
        "redmine": {
          "command": "redmine",
          "args": ["mcp", "serve", "--profile", "work"]
        }
      }
    }
    ```

    若要启用写入，可新增一个条目或替换 `args`：

    ```json
    {
      "mcpServers": {
        "redmine-write": {
          "command": "redmine",
          "args": ["mcp", "serve", "--profile", "work", "--enable-writes"]
        }
      }
    }
    ```
  </TabItem>

  <TabItem label="Cursor">
    创建 `~/.cursor/mcp.json`（全局）或在项目根目录创建 `.cursor/mcp.json`：

    ```json
    {
      "mcpServers": {
        "redmine": {
          "command": "redmine",
          "args": ["mcp", "serve", "--profile", "work"]
        }
      }
    }
    ```
  </TabItem>

  <TabItem label="Zed">
    在 `settings.json` 中添加 context server（按 `cmd+,` 后选择 “Open Settings”）：

    ```json
    {
      "context_servers": {
        "redmine": {
          "command": {
            "path": "redmine",
            "args": ["mcp", "serve", "--profile", "work"]
          }
        }
      }
    }
    ```
  </TabItem>

  <TabItem label="GitHub Copilot">
    在 VS Code 中，于工作区创建 `.vscode/mcp.json`（或添加到用户设置）：

    ```json
    {
      "servers": {
        "redmine": {
          "type": "stdio",
          "command": "redmine",
          "args": ["mcp", "serve", "--profile", "work"]
        }
      }
    }
    ```

    在 Copilot Chat 中启用 MCP，并从工具选择器中选择 `redmine` 服务器。
  </TabItem>
</Tabs>

### 故障排查

<Steps>

1. **宿主报告 “command not found”。** 生成的进程继承宿主的 `PATH`，这通常与你的 shell 不同。请在 `command` 字段中使用绝对路径（`which redmine`）。

2. **工具列表为空或缺少修改类工具。** 未传入 `--enable-writes`，或宿主缓存了旧的 `tools/list`。修改参数后请重启宿主。

3. **401 / “no profile” 错误。** 当前没有登录的配置文件，或 `--profile <name>` 指向的配置文件不存在。运行 `redmine auth list` 进行确认。

</Steps>
