## create

```bash
redmine versions create --name <name> [flags]
```

别名：`new`。

| 标志 | 描述 |
|------|------|
| `--project`         | 项目标识符——默认回退至活跃配置文件的默认项目 |
| `--name`            | 版本名称（必填） |
| `--status`          | 状态：`open`、`locked`、`closed` |
| `--sharing`         | 共享范围：`none`、`descendants`、`hierarchy`、`tree`、`system` |
| `--due-date`        | 截止日期（`YYYY-MM-DD`） |
| `--description`     | 版本描述 |
| `--wiki-page-title` | 关联的 Wiki 页面标题 |
| `-o, --output`      | 输出格式 |

```bash
redmine versions create --project myproject --name 1.2.0 --due-date 2026-06-30
```

## list

```bash
redmine versions list --project <identifier> [flags]
```

| 标志 | 描述 |
|------|------|
| `--project`    | 项目标识符——默认回退至活跃配置文件的默认项目 |
| `--status`     | 按状态过滤：`open`、`locked`、`closed` |
| `--limit`      | 最大结果数 |
| `--offset`     | 结果偏移量 |
| `-o, --output` | 输出格式 |

```bash
redmine versions list --project myproject --status open
```

## get

```bash
redmine versions get <id-or-name> [flags]
```

别名：`show`、`view`。接受数字 ID 或版本名称。

| 标志 | 描述 |
|------|------|
| `--project` | 项目标识符（用于名称解析；默认回退至默认项目） |

<Aside type="note">
  通过名称解析版本时，必须已知项目信息。CLI 优先使用 `--project`，否则使用活跃配置文件中的 `default_project`。
</Aside>

## update

```bash
redmine versions update <id-or-name> [flags]
```

别名：`edit`。接受数字 ID 或版本名称。

| 标志 | 描述 |
|------|------|
| `--project`         | 项目标识符（用于名称解析；默认回退至默认项目） |
| `--name`            | 新版本名称 |
| `--status`          | 新状态：`open`、`locked`、`closed` |
| `--sharing`         | 新共享范围：`none`、`descendants`、`hierarchy`、`tree`、`system` |
| `--due-date`        | 新截止日期（`YYYY-MM-DD`） |
| `--description`     | 新版本描述 |
| `--wiki-page-title` | 新关联的 Wiki 页面标题 |

```bash
redmine versions update 42 --status closed
redmine versions update 1.2.0 --project myproject --due-date 2026-07-15
```

## delete

```bash
redmine versions delete <id-or-name> [flags]
```

别名：`rm`。接受数字 ID 或版本名称。

| 标志 | 描述 |
|------|------|
| `--project` | 项目标识符（用于名称解析；默认回退至默认项目） |
| `--force`   | 删除前不询问确认 |

```bash
redmine versions delete 42
redmine versions delete 1.2.0 --project myproject --force
```

<Aside type="caution">
  删除版本是破坏性操作。未使用 `--force` 时，CLI 会在发送删除请求前要求确认。
</Aside>
