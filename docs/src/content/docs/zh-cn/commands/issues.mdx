## list

```bash
redmine issues list [flags]
```

| 标志 | 描述 |
|------|------|
| `--project`     | 按项目过滤（名称、标识符或 ID） |
| `--tracker`     | 按跟踪类型过滤（名称或 ID） |
| `--status`      | 按状态过滤：`open`、`closed`、`*`、名称或 ID |
| `--assignee`    | 按指派人过滤：`me`、名称或 ID |
| `--version`     | 按目标版本过滤（名称或 ID） |
| `--sort`        | 排序方式，例如 `updated_on:desc` |
| `--include`     | 包含关联数据：`attachments`、`relations` |
| `--attachments` | `--include attachments` 的简写 |
| `--relations`   | `--include relations` 的简写 |
| `--limit`       | 最大结果数（0 表示全部） |
| `--offset`      | 分页偏移量 |
| `-o, --output`  | 输出格式：`table`、`wide`、`json`、`csv` |

<Tabs syncKey="issues-list">

## get

```bash
redmine issues get <id> [flags]
```

| 标志 | 描述 |
|------|------|
| `--include`    | 包含关联数据：`journals`、`children`、`relations` |
| `--journals`   | `--include journals` 的简写 |
| `--children`   | `--include children` 的简写 |
| `--relations`  | `--include relations` 的简写 |
| `-o, --output` | 输出格式 |

## create

```bash
redmine issues create [flags]
```

| 标志 | 描述 |
|------|------|
| `--project`         | 项目（名称、标识符或 ID）——默认回退至活跃配置文件的默认项目 |
| `--subject`         | 工单标题 — <Badge text="必填" variant="danger" /> |
| `--tracker`         | 跟踪类型（名称或 ID） |
| `--status`          | 状态（名称或 ID） |
| `--priority`        | 优先级（名称或 ID） |
| `--assignee`        | 指派人：`me`、名称或 ID |
| `--description`     | 工单描述 |
| `--parent`          | 父工单 ID |
| `--category`        | 类别（名称或 ID） |
| `--version`         | 目标版本（名称或 ID） |
| `--estimated-hours` | 预估工时 |
| `--private`         | 标记为私有 |
| `--attach`          | 要附加的文件路径（可重复使用） |

```bash
redmine issues create --project myproject --subject "Add search" --tracker Feature --priority High

redmine issues create --project myproject --subject "Bug report" \
  --attach /path/to/screenshot.png --attach /path/to/log.txt
```

## update

```bash
redmine issues update <id> [flags]
```

接受与 `create` 相同的标志（`--project` 除外），并额外支持：

| 标志 | 描述 |
|------|------|
| `--note`       | 添加评论 |
| `--done-ratio` | 完成度百分比（0-100） |
| `--attach`     | 要附加的文件路径（可重复使用） |

```bash
redmine issues update 123 --note "Fixed the bug" --attach /path/to/fixed_code.patch
```

## close

```bash
redmine issues close <id> [flags]
```

| 标志 | 描述 |
|------|------|
| `--note` | 添加关闭评论 |

## reopen

```bash
redmine issues reopen <id> [flags]
```

## assign

```bash
redmine issues assign <id> <user-id-or-name> [flags]
```

用户参数接受数字 ID、登录名、全名或 `me`。

## comment

```bash
redmine issues comment <id> --message "Your comment" [flags]
```

## delete

<Badge text="危险操作" variant="caution" />
```bash
redmine issues delete <id> [flags]
```

| 标志 | 描述 |
|------|------|
| `-f, --force` | 跳过确认提示 |

<Aside type="caution">
  删除操作不可逆，无法通过 CLI 撤销。除非确有必要，否则请优先使用 `close`。
</Aside>

## open

```bash
redmine issues open <id>
```

根据已配置的 Redmine 服务器构造工单 URL，并在默认浏览器中打开。

```bash
redmine issues open 123   # → https://redmine.example.com/issues/123
```

## browse

```bash
redmine issues browse [flags]
```

打开工单的交互式终端浏览器。接受与 [`list`](#list) 相同的过滤标志。
